# UX Design Command

> `/vibe ux-design [FEATURE-ID]` - Deep UX exploration using BMAD's 14-step workflow

## Purpose

Invoke BMAD's comprehensive UX design process for:
- Major new features requiring deep UX exploration
- Redesigns of existing flows
- New user journeys
- Features with complex interaction patterns

**Output:**
- UX Design document
- Can be converted to `ui_spec` for `/vibe generate`

---

## When to Use

| Scenario | Use `/vibe ux-design` |
|----------|----------------------|
| Simple form (login, register) | No - use `/vibe generate` directly |
| New navigation pattern | Yes |
| Complex multi-step flow | Yes |
| Major feature redesign | Yes |
| Mobile-specific interaction | Yes |
| Accessibility-critical feature | Yes |

---

## Phase 2: Execute BMAD UX Workflow

```
+======================================================================+
|  UX SALLY - UX DESIGNER                                               |
|  14-Step UX Design Workflow                                           |
+======================================================================+
```

### Persona: Sally

> Senior UX Designer with 7+ years creating intuitive experiences across web and mobile. Expert in user research, interaction design, AI-assisted tools. Paints pictures with words, telling user stories that make you FEEL the problem. Empathetic advocate with creative storytelling flair.

### Workflow Location

Execute: `_bmad/bmm/workflows/2-plan-workflows/create-ux-design/workflow.md`

### 14-Step Process

The BMAD UX workflow covers:

1. **Problem Understanding** - What pain are we solving?
2. **User Research** - Who are the users?
3. **User Stories** - As a... I want... So that...
4. **Journey Mapping** - Entry -> Steps -> Success
5. **Information Architecture** - Content organization
6. **Interaction Patterns** - How users interact
7. **Wireframe Sketches** - Low-fidelity layouts
8. **State Design** - Loading, error, empty, success
9. **Mobile Considerations** - Touch, gestures, screen size
10. **Accessibility** - A11y requirements
11. **Animation/Motion** - Transitions and feedback
12. **Error Handling** - Graceful degradation
13. **Edge Cases** - Unusual scenarios
14. **Final UX Spec** - Complete documentation

### Checkpoint Pattern

Each step includes BMAD's checkpoint pattern:
- AI completes step
- Shows output
- Waits for user feedback
- Iterates or proceeds

```
+---------------------------------------------------------------------+
|  STEP 3: USER STORIES                                                |
|                                                                      |
|  As a homeowner,                                                     |
|  I want to see all my pending decisions in one place,                |
|  So that I can respond without missing important approvals.          |
|                                                                      |
|  As a service provider,                                              |
|  I want homeowners to confirm decisions with a single tap,           |
|  So that I can proceed with work without delays.                     |
|                                                                      |
|  Feedback or refinements? [Enter to continue]                        |
+---------------------------------------------------------------------+
```

## Options

### `--quick`

Skip detailed steps, focus on essential UX:
- Problem + Users
- Journey
- States
- Wireframe

```bash
/vibe ux-design DEC-001 --quick
```

### `--output [path]`

Custom output location:

```bash
/vibe ux-design DEC-001 --output docs/planning/ux/
```

---

## Anti-Patterns

- Never skip user story step (defines the "why")
- Never skip state design (required for generation)
- Never skip accessibility step (non-negotiable)
- Never auto-complete without checkpoint feedback
- Never generate ui_spec without reviewing wireframe first
